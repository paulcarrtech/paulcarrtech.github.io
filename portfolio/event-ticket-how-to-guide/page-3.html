Ticket Types

Ticket types define the access permissions of printed tickets. Every admission price point (e.g., Adult, Senior, Student) for an individual tour typically share the same ticket type.

Follow the steps below to reuse or create a ticket type.

Log in to Centaman

  1. From your desktop, open Remote Desktop Viewer.
  2. Enter your credentials when prompted.
  3. Click Connect.
  4. Enter your password when prompted.
  5. From the grey text box, press any key to continue.

The next screen with orange buttons is your launch pad.

Manage Ticket Types in Enterprise Management Studio

  1. From the Centaman launch pad, click on Enterprise Management Studio.
  2. Enter your credentials to log in.
  3. From the left sidebar, click Access Control.
  4. From the left sidebar again, click Ticket Types.
  5. Near the top of the main panel, click on the category Ticket Type or Ticket Name to sort the list of ticket types.
Reuse a Ticket Type

Every admission price point (e.g., Adult, Senior, Student) for an individual tour shares a ticket type. Ticket types can be reused between related tours.

Note
Each ticket type defines how a physical ticket behaves if scanned: how many times it can be scanned for entry and how often; how long it remains valid; and whether in and outs are allowed. As of this writing, such ticket behaviors are not critical to our operations.

Here are some common ticket types that might suit your needs:

Select a Ticket Type to Reuse if Applicable
  • If an existing ticket type suits your needs, skip ahead to the stock buttons section. You will select a ticket type to reuse later.
  • If you need to create a new ticket type now, proceed to the following section.
Create a Ticket Type
  1. From below list of ticket types, click New Ticket Type to open the New Ticket Type form.
  2. Complete the New Ticket Type form:
    Field Name Action
    Ticket Type Enter a 4-digit number (starting with 6 for consistency) that has not yet been assigned to an existing ticket type.
    Description Name Give the ticket type a name (e.g. Public Program – Workshop)
    Multi Entry Check this box.
    Entry Limit Set this to 5 (arbitrarily). This field defines how many times the ticket can be scanned.
    Passback Timer Set this to 5 (arbitrarily). This field defines how many minutes can pass before the ticket can be scanned again.
    Expiry Time Check this box.
    Expirey Interval Set this field to the length in minutes of the event plus any post-event mingling or exhibition exploration. Most events run between 120–420 minutes.
    Fixed Expiry Leave this box blank.
    Days Valid Leave this field blank.
    Scan Required Check this box.
    Is Admission Check this box.
  3. Click Save at the end of the form.
  4. Click New Ticket Type Details to open the New Ticket Type Details form.
  5. Complete the New Ticket Type Details form:
    Field Name Action
    Entry Check this box.
    Entry Limit Set this to 5 (arbitrarily).
    Exit Leave this box blank.
    Exit Limit Leave this field blank.
    Passback Timer Set this to 5 (arbitrarily).
  6. Click Save at the end of the form.
  7. Close Enterprise Management Studio.
